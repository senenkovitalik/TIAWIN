SSH

Secure Shell (SSH) це протокол, який забезпечує безпечне з'єднання віддаленого доступу до мережевих пристроїв. Зв'язок між клієнтом і сервером шифрується як в SSHv1 так і в SSHv2. Краще використовувати SSHv2, тому що використовує більш вдосконалений алгоритм шифрування безпеки.

У цьому пості обговорюється, як налаштувати і налагодити SSH на Cisco маршрутизаторах і комутаторах, які працюють на ПЗ Cisco IOS®, яке підтримує SSH.

Cisco IOS образ, що використовується має бути k9 (crypto) образ з підтримкою SSH. Наприклад c3750e-universalk9-tar.122-35.SE5.tar є K9 (crypto) образом.

!--- Step 1: Configure the hostname if you have not previously done so.

hostname carter

!--- The aaa new-model command causes the local username and password on the router
!--- to be used in the absence of other AAA statements.

aaa new-model
username cisco password 0 cisco

!--- Step 2: Configure the DNS domain of the router.

ip domain-name rtp.cisco.com

!--- Step 3: Generate an SSH key to be used with SSH.

crypto key generate rsa
ip ssh time-out 60
ip ssh authentication-retries 2

!--- Step 4: By default the vtys' transport is Telnet. In this case,
!--- Telnet is disabled and only SSH is supported.

line vty 0 4
transport input SSH

!--- Instead of aaa new-model, you can use the login local command.
